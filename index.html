How it works under the hood

Every demo shares one tiny wrapper around the browser's built-in LanguageModel API — the same pattern shown in the reference implementation. Feature-detect, create a session, stream the response token by token.

// the real Prompt API — mirrored across all 10 demos
const lm = window.LanguageModel;
const status = await lm.availability();        // 'available' | 'downloadable' | …

const session = await lm.create({
  initialPrompts: [{ role: 'system', content: systemPrompt }],
  temperature: 0.4, topK: 3,
  monitor(m){ m.addEventListener('downloadprogress', e => show(e.loaded/e.total)); }
});

const stream = session.promptStreaming(userText, { responseConstraint: schema });
for await (const chunk of stream) { output += chunk; render(output); }
💡

About "Demo mode": these demos run the real on-device model when your browser supports it (Chrome with the built-in AI / Prompt API enabled). On any other browser they automatically fall back to a faithful simulated stream so you can still experience the full UX — each page shows a badge telling you which mode it's in. Nothing is ever sent to a server in either case.
